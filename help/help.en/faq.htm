Frequently Asked Questions
Q.   EZ MP3 Creator doesn't find a cdrom to use, the only item in the cdrom list is 'none', but I have a cdrom device certainly, and it works fine. How can I get the program to recognize my cdrom?
A.   EZ MP3 Creator uses ASPI layer to access cdroms. This problem is usually caused by problems of ASPI driver. To fix it, just re-install ASPI driver. You can get ASPI driver from Adaptec. EZ MP3 Creator doesn't support USB/FIREWIRE devices in Windows 95/98/ME.
     
Q.   I only seem to be able to extract and convert odd tracks from a cd, am I doing something wrong?
A.   This is the limitation of evaluation version, please order full version to remove the limitation.
     
Q.   I get a track with a strike line, and I can't select it, am I doing something wrong?
A.   This track is not an audio track, so it is marked with strike line, and it can not be selected.
     
Q.   I only get the last track when ripping from cd to any format, am I doing something wrong?
A.   It should be something wrong with the filename format in the options page, which causes that every track uses the same file name, so each track is overwritten by the next one. For example, you use <title> as filename format, but you have set titles of all tracks to the same one. It is highly recommended to include the <track> tag (it means track number) in filename format.
     
Q.   After executing EZ MP3 Creator, the title of my cd became some strange characters, why?
A.   Perhaps you have downloaded the wrong cd information with EZ MP3 Creator, because windows system uses the same information as EZ MP3 Creator. The solution is to download the correct cd information with EZ MP3 Creator, or delete the wrong section in file windows\cdplayer.ini.
